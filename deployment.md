# Deployment (Vercel)

The site is a Next.js 16 app in [frontend/](frontend/). It reads pre-built JSON artifacts from [data/published/](data/published/) — a directory that sits **outside** the Next.js root — via [frontend/src/lib/data.ts](frontend/src/lib/data.ts). Every dynamic route uses `generateStaticParams` with `revalidate = 86400`, so pages are statically generated at build *and* re-generated by serverless functions at request time after 24 hours. Both phases need access to `data/published/`.

## One-time setup

1. Push the repo to GitHub.
2. At https://vercel.com/new, import the repo.
3. **Root Directory:** set to `frontend`. Vercel still clones the full repo, so `../data/published` is reachable from the build.
4. **Framework preset:** Next.js (auto-detected).
5. **Build command / Output:** leave as defaults.
6. **Environment variables** (Settings → Environment Variables):
   - `NEXT_PUBLIC_SITE_URL=https://collegegradanalyst.com`
   - copy anything else from your local `.env` that the app reads.
   > The canonical domain is `collegegradanalyst.com`. The legacy `collegeoutcomeanalyst.com` value can leak via this env var on the host — set it explicitly.
7. **Domains** (Settings → Domains): add `collegegradanalyst.com` and `www.collegegradanalyst.com`, then follow the DNS instructions Vercel shows.

## Data delivery: split between bundle and CDN

[frontend/src/lib/data.ts](frontend/src/lib/data.ts) uses two strategies, because bundling all of `data/published/` into every serverless function pushes past Vercel's 300 MB function cap:

- **Small artifacts** (`home.json`, `rankings.json`, `methodology.json`, `roi_constants.json`, `_manifest.json`, `state/**`) are read from disk via `fs.readFileSync`. They're bundled into each function by `outputFileTracingIncludes` in [frontend/next.config.ts](frontend/next.config.ts).
- **Big trees** (`program/`, `institution/`, `city/`) are fetched over HTTP at request time from jsDelivr, which serves them straight from the public GitHub repo. The base URL defaults to `https://cdn.jsdelivr.net/gh/rwongphoto/college-study-data@main/data/published` and can be overridden with `NEXT_PUBLIC_DATA_CDN_BASE` for staging or branch previews. Responses are cached via Next's `revalidate: 86400`.

[frontend/next.config.ts](frontend/next.config.ts) sets `outputFileTracingRoot` one level up so the includes resolve, and lists exactly the small files to bundle:

```ts
outputFileTracingIncludes: {
  "/**": [
    "../data/published/home.json",
    "../data/published/rankings.json",
    "../data/published/methodology.json",
    "../data/published/roi_constants.json",
    "../data/published/_manifest.json",
    "../data/published/state/**",
  ],
},
```

[frontend/scripts/prebuild.mjs](frontend/scripts/prebuild.mjs) does two jobs on every build:

1. Walks `data/published/` to emit `_manifest.json` (the directory listing the sitemap and `listCities` / `listInstitutions` rely on).
2. On Vercel/CI only, deletes `program/`, `institution/`, and `city/` from the build workspace **after** reading them. Next's file tracer otherwise keeps finding them and bundling them anyway, busting the 300 MB cap. The trees stay in GitHub, so jsDelivr keeps serving.

Local dev uses [frontend/scripts/predev.mjs](frontend/scripts/predev.mjs) instead, which emits the same manifest plus symlinks `frontend/public/data/{program,institution,city}` → `data/published/...` so the dev server can serve those files locally without going to the CDN. Vercel does not run this script.

If you add a new top-level artifact to `data/published/`, decide which side it belongs on: small + always-needed → add to `outputFileTracingIncludes` and a `loadX` that uses `readJSON`; large or per-entity → add a `fetchJSON` loader and let jsDelivr serve it.

## Day-to-day

- **Deploys:** every push to `main` deploys to production. PR branches get preview URLs.
- **Refreshing data:** rebuild `data/published/` locally (or in CI), commit, push. ISR will pick up changes within 24h on already-rendered pages, or instantly for new ones.
- **Force a rebuild without code changes:** Vercel dashboard → Deployments → … → Redeploy.

## Sanity checks after first deploy

- `https://collegegradanalyst.com/sitemap.xml` returns the expected URLs.
- A state page (e.g. `/state/california/`) renders with rankings and the ROI calculator.
- An institution page renders and the `<RoiCalculator>` is present.
- JSON-LD `@graph` is present in `<head>` and uses the canonical domain.
